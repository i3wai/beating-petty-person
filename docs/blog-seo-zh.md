## 1. 核心原則

### EN 和 ZH blog 是兩套獨立內容體系

- **EN blog**：攻英文高搜索量長尾詞（curse, hex, black magic, voodoo）— 神秘學市場
- **ZH blog**：攻中文文化+命理關鍵詞（打小人, 驚蟄, 太歲, 去霉運）— 民俗信仰市場
- **不做直接翻譯**。同一概念可以有對應主題，但標題、內容、角度、文化引用完全獨立
- **URL 可以對應**（例如 EN `/blog/history-of-villain-hitting` ↔ ZH `/blog/驚蟄打小人攻略`），但內容各自為戰

### 為什麼不翻譯

1. 搜索意圖不同：EN 用戶搜 "how to curse someone"，ZH 用戶搜 "驚蟄打小人"
2. 文化語境不同：EN 市場視打小人為異國神秘體驗；ZH 市場視之為日常民俗
3. 競爭格局不同：EN 競爭對手是 spell/witchcraft 網站；ZH 競爭對手是命理博客和維基百科
4. 關鍵詞完全不同：兩套語言的搜索量、KD、搜索意圖無交集

## 2. 被拒絕的關鍵詞與理由

以下關鍵詞在 Agent 討論中被提出，經批判後被否決：

| 關鍵詞 | 月搜量 | 拒絕理由 |
|--------|--------|----------|
| **生氣怎麼辦** | 高 | 搜索意圖是心理調節，不是詛咒/民俗。流量高但轉換為零，且會混淆品牌定位 |
| **發洩情緒方法** | 高 | 同上。用戶在找心理學答案，不是打小人。我們的定位是「詛咒儀式」，不是「情緒管理」 |
| **轉運方法** | 中高 | 太泛。可以作為 Cluster B 的輔助關鍵詞，但不值得獨立成篇。跟太多風水/算命網站競爭 |
| **職場小人** | 5,000+ | 搜索意圖是「如何應對」，不是詛咒。但可作為打小人的語境關鍵詞，不必獨立文章 |

### 否決邏輯

BeatPetty 的品牌定位是 **「詛咒儀式」**，不是心理諮詢、不是職場建議、不是風水教室。所有內容必須服務於：

1. 強化「打小人 = 詛咒儀式」的品牌認知
2. 引導用戶完成線上儀式（轉換終點）
3. 建立在「詛咒/民俗」領域的 topical authority

任何偏離這個定位的關鍵詞，即使流量再高，都是噪音。

## 6. 與 EN Blog 的反蠶食策略

### 關鍵詞不重疊

- EN 文章用英文關鍵字（"black magic", "hex"），ZH 文章用中文關鍵字（"打小人", "去霉運"）
- Google 按語言分開索引，hreflang 已正確設置
- 不存在 EN/ZH 關鍵詞自我蠶食

### ZH-TW vs ZH-Hans 差異化

- 內容主體相同（接受重疊）
- 文化語境調整：
  - zh-TW：香港用語（鵝頸橋、驚蟄）、繁體習慣
  - zh-Hans：通用大陸用語、簡體習慣
- 如需防重複內容，使用 canonical 標籤

### URL 策略

- ZH 文章 slug 使用中文字符（如 `/blog/打小人完整攻略`）
- 也可以使用拼音 slug（如 `/blog/da-siu-yan-guide`）— 視 Google 索引表現決定
- **建議先用中文 slug**，因為中文用戶搜索中文關鍵字，URL 中的中文有助於 SERP 點擊率

## 8. ZH Blog 文案調性指南

### 語言風格

- **不回避「詛咒」「小人」** — 這是產品核心，也是搜索詞
- **正統文化語氣** — 「正宗」「傳統」「百年」— 不是搞笑，不是學術，是敬畏
- **暗示超自然** — 「萬一呢？」— 不承諾效果，但暗示可能性
- **繁體中文（zh-TW）**：粵語/香港文化用語（驚蟄、鵝頸橋）
- **簡體中文（zh-Hans）**：大陸通用用語

### CTA 文案

- ✓「親自體驗打小人儀式」
- ✓「開始打小人」
- ✓「免費體驗正宗詛咒儀式」
- ✗「了解我們的服務」
- ✗「試試我們的工具」

### 打小人段落（每篇 50-80 字）

每篇文章都要有一段打小人的連結段落，但切入角度不同：

- ZH1：「打小人是什麼？從這篇文章到實際體驗」

### Cluster 代號擴展

在 `src/lib/content.ts` 中，`cluster` 欄位需要支援 ZH 特定的集群標識：

```typescript
// 方案 A：EN 和 ZH 共用 "A"/"B"/"C" 代號（推薦）
// ZH Cluster A → cluster: "A"（與 EN Cluster A 同代號，但語意不同）
// 好處：代碼不動，Blog Hub 按語言分開渲染

// 方案 B：新增 "D"/"E"/"F" 代號
// ZH Cluster A → cluster: "D"
// 好處：明確區分 EN 和 ZH 集群
// 壞處：需改 Blog Hub 組件支援更多集群
```

**建議使用方案 A**：EN 和 ZH 的 blog hub 是不同頁面（/en/blog vs /zh-TW/blog），各自渲染各自的集群，代號相同不衝突。Cluster 標題用 i18n key 解決。

### 3-Agent 討論參與者
1. **SEO 策略師**：負責關鍵詞研究、集群架構、搜索意圖分析

### 主要爭論點與結論

1. **是否擴展到「去霉運」「轉運」等泛命理詞？**
   - 結論：有限度擴展。去霉運、犯太歲可接受（與打小人在同一民俗信仰體系），但「生氣怎麼辦」「發洩情緒」被否決（意圖不匹配）

2. **4 集群 vs 3 集群？**
   - 原提案 4 集群（打小人核心 / 去霉運與轉運 / 職場與情緒 / 驚蟄與季節）
   - 結論精簡為 3 集群：刪除「職場與情緒」（意圖不匹配），「驚蟄」併入打小人核心

3. **太歲和鬼月的發現**
   - 原提案完全沒有提到太歲和鬼月
   - 這是 Agent 討論中最重要的新增：兩者都是持續性搜索需求，且完美對齊打小人定位

4. **第一篇寫什麼？**
   - 結論：驚蟄打小人攻略。低 KD、高意圖、品牌最強連結

### Elon 批判意見

- 「生氣怎麼辦」被否決是正確的。我們是詛咒網站，不是心理諮詢平台。
- 太歲和鬼月的加入是整個討論中最有價值的發現。這兩個關鍵詞群有持續性流量，且文化語境完全對齊。
- 3 集群優於 4 集群。少即是多——每個集群更聚焦，內部連結更有效。
- 中文 slug 是正確選擇。Google 對中文 URL 的處理已經很成熟，SERP 點擊率更高。
